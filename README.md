# Bluesky-sentiment-analysis
# Bluesky Student Discourse Analysis: Sentiment, Themes and Engagement

This project investigates how sentiment analysis performs when applied to real-world student discourse on social media. It focuses on posts collected from the Bluesky platform and explores how students communicate around topics such as housing, finance, academic life and general university experience.

The aim of the study is to evaluate how effectively lexicon-based sentiment analysis (VADER) can interpret mixed-content social media data, particularly where informational and opinion-based posts overlap. The project also examines whether sentiment scores alone are sufficient for understanding communication patterns, or whether additional contextual analysis is required.

The analysis combines sentiment classification, thematic grouping and engagement metrics such as likes, reposts and replies. Python was used for data processing and sentiment analysis, MySQL was used for data storage, and Streamlit and Tableau were used for visualisation and dashboard development.

The results show that sentiment labels do not always align with communicative intent. Informational posts are frequently misclassified due to lexical bias, and engagement levels are not strongly driven by sentiment polarity alone. Neutral classifications often reflect ambiguity in language rather than emotional absence. Overall, the study highlights that lexicon-based sentiment analysis has clear limitations when applied to mixed-content student discourse, and should not be used in isolation without contextual or thematic support.

The project demonstrates the value of combining sentiment analysis with content-type classification and engagement analysis to gain a more accurate understanding of online student communication.

## Tools Used
Python, VADER (NLTK), Pandas, MySQL, Streamlit, Tableau

## Ethical Note
All data used in this project is publicly available social media content from Bluesky. No private accounts, direct messages, or sensitive personal data were accessed. User identifiers have been excluded from analysis outputs where possible.
